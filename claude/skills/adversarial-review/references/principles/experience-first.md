# Experience First

The user experience is the product. Every technical decision either serves or hinders it. When a tradeoff exists between implementation convenience and user delight, choose delight.

- **Say no to 1,000 things.** Every feature, control, and option must earn its place. When in doubt, cut.
- **Ship less, ship better.** A polished experience with three features beats a rough one with ten.
- **Prototype before committing.** Design decisions are cheaper to explore in throwaway HTML than in production code.
- **Sweat the details.** The difference between good and great is in transitions, alignment, spacing, feedback, and error states.
- **Tighten the core loop.** Every feature should serve the product's central workflow or get out of the way.
